łarge \#19 FochBot — Autonomiczny Asystent AI\\[4pt]

FastAPI · Mikrousługi · BCRYPT ·

Warstwa pośrednia LLM · Python · 2024–2026

FochBot to projekt rozwijany przez trzy generacje architektoniczne —

od prostego klasyfikatora intencji do pełnoprawnej pośredniej

między u\.zytkownikiem a modelem językowym.

Dziś FochBot nie jest klientem API ani frontendem do LLM:

jest mózgiem systemu, który utrzymuje ciągłość, pamięć

i decyzyjność niezale\.znie od stanu modelu językowego.

V1 — Klasyfikator intencji i NLP

Pierwsza wersja: lokalny asystent oparty na klasyfikacji intencji.

Architektura V1:

  • intents.json — baza intencji z przykładowymi zdaniami
  • core/nlp.py — tokenizacja i czyszczenie tekstu
  • core/classifier.py — klasyfikator intencji (logika ML),

training\_data.pkl

  • core/responder.py — generowanie odpowiedzi na podstawie

sklasyfikowanej intencji

  • interface/cli.py — interfejs tekstowy;

interface/web.py — opcjonalny front webowy (Flask\,/ Gradio)

  • core/arr.py — zarezerwowany pod integrację z analizą

rytmu MR (slot na dalszy rozwój)

Logi rozmów i debug zapisywane lokalnie; model działa w pełni offline.

V2 — Mikrousł

Pełna przebudowa na architekturę mikrousług.

Dwie niezale\.zne usługi komunikujące się po sieci wewnętrznej:

Usługa Odpowiedzialność
API :8000 Logika biznesowa: chat, odpowiedzi bota, endpointy dla klientów
tokeny urządzeń, role

Przepływ autoryzacji:

Ka\.zde \.ządanie do :8000/chat przechodzi przez require\_auth() —

mostek access\_control.py wysyła POST /check\_access do :8001,

który sprawdza sesję w RAM i zwraca allowed: true/false

wraz z danymi u\.zytkownika.

Baza danych JSON (pliki lokalne):

  • users.json — u\.zytkownicy, hasze BCRYPT, role

(admin / user / device)

  • devices.json — autoryzowane urządzenia, whitelist IP, tokeny
  • data\_manager.py — jedyny moduł z prawem zapisu;

atomowy zapis odporny na awarię w trakcie operacji

Zarządzanie przez CLI (user\_tool.py):

dodawanie u\.zytkowników, nadawanie uprawnień, dezaktywacja kont.

Frontend: fochbot-frontend-v2. Dostarczono w trzech fazach (phase1–3).

V3 — Warstwa po\'s

Zasada fundamentalna V3:

Model LLM (GPT, Claude itp.) to neuron — jednorazowa jednostka obliczeniowa.

Otrzymuje impuls, generuje wyjście i gaśnie.

Nie utrzymuje stanu, nie zna swojego poprzedniego wywołania.

FochBot jest mózgiem: utrzymuje ciągłość,

pamięć i decyduje, kiedy i jak wywołać LLM.

Architektura V3 składa się z dwóch niezale\.znych warstw:

Warstwa A — Mapa Pamięci (kontekst):

Trzypoziomowe archiwum na dysku —

surowe sesje (session\_*.json),

indeks (kompaktowy katalog dla LLM)

i fragmenty wiedzy (bloki wczytywane selektywnie per zapytanie).

LLM zawsze dostaje tylko tyle kontekstu, ile potrzebuje — bez pełnej historii.

Warstwa B — Kontroler Autonomiczny (decyzyjność):

Podejmuje decyzje lokalne bez LLM (routing, ocena kompletności,

budowanie pakietu zapytania), następnie wywołuje LLM jako narzędzie.

Ocenia wynik, ewentualnie inicjuje ponowne wywołanie lub eskalację.

Logika ciągłości po stronie FochBota, nie po stronie modelu językowego.

Stos techniczny V3:

FastAPI backend (app/backend/), frontend (app/frontend/),

zarządzanie stanem (state/), pełna dokumentacja techniczna

(FochBot\_V3\_Dokumentacja.tex\,/\,.pdf\,/\,.docx).

Stan projektu (aktywny R\&D, 2024–2026):

  • V1 — klasyfikator intencji, NLP offline, logowanie rozmów
  • V2 — mikrousługi (FastAPI), BCRYPT, sesje, role,

frontend, dostarczony w 3 fazach

  • V3 — warstwa pośrednia LLM: Mapa Pamięci + Kontroler Autonomiczny;

dokumentacja techniczna V3 (PDF\,/ DOCX\,/ LaTeX)

  • Projekt otwarty: integracja z analizą MR-AI (projekt \#20),

rozwój warstwy B w kierunku pełnej autonomii
